- WLS 的优化目标只是把输入向量当成一个实数序列，直接进入 chi² = Σ (p_obs - p_exp)² / √(p_obs + 1)（python/qtomography/domain/reconstruction/wls.py (lines 270-284)）。只要归一化时分组总和不为 0，它就能得到一个可导的目标函数，并且在每一步都通过 Cholesky+DensityMatrix 投影把 ρ 保持为物理态。即便某些 p_obs 是负数，算法本质上是在做“平方误差拟合”——会强制 p_exp 接近这些负值，但最终密度仍 PSD，所以偏差主要体现在拟合目标与真实数据的落差，而不是迭代数值爆炸。
- RρR strict 的推导假设 f_j ≥ 0 且 Σ_j f_j = 1，因为在迭代里会计算 r_j = f_j / q_j 和 ll = Σ f_j log q_j（python/qtomography/domain/reconstruction/rhor_strict.py (lines 295-343)）。当 f_j 出现负值时，r_j 立刻变成负数，使得 R = Σ r_j Ē_j 可能出现负特征值，进而让 σ ← R σ R 带入错误的方向、触发频繁的 trace reset（python/qtomography/domain/reconstruction/rhor_strict.py (lines 313-338)）。这个算法相当于 EM-like 的概率更新，一旦输入不满足“概率”假设，单调性证明就失效，很容易陷入震荡或卡在离真解很远的位置。
- 总结：WLS 只是把输入视为要拟合的数值并通过全局优化找到“最接近”的 ρ，因此对符号异常相对更鲁棒；RρR 则把输入直接当作条件概率参与乘法/对数迭代，负值会破坏其物理前提，使得迭代方向、对数似然都不再有意义，所以表现更差。根本解决办法是在进入任何重构器之前就保证测量频率非负并按组归一化，否则这类物理约束算法都无法得到可信结果。
